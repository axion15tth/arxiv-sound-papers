# 要件定義書：arXiv Sound Papers Viewer

## 1. プロジェクト概要

### 1.1 目的
音響分野の研究者向けに、arXivに投稿された音響関連論文を日本語タイトル付きで閲覧でき、ブックマーク・タグ管理ができるWebアプリケーションを開発する。

### 1.2 背景
- Twitter（X）の@ArxivSoundアカウントでは音響関連論文が英語で共有されている
- 日本人研究者にとって英語タイトルの閲覧は負担が大きい
- 論文を分野別（音声、音楽、信号処理等）に整理して管理したいニーズがある

### 1.3 主要ユーザー
- 音響分野の研究者（学生、教員、エンジニア）
- 複数の研究分野に関心を持つ研究者

---

## 2. 機能要件

### 2.1 論文一覧表示機能（優先度：高）

#### 2.1.1 表示項目
- 論文タイトル（英語・オリジナル）
- 論文タイトル（日本語・翻訳）
- 著者名
- 投稿日
- arXivカテゴリ（cs.SD, eess.AS, cs.AI等）
- arXiv論文ページへのリンク

#### 2.1.2 データ取得仕様
- **取得元**: arXiv API
- **対象カテゴリ**:
  - cs.SD (Sound)
  - eess.AS (Audio and Speech Processing)
  - cs.AI (Artificial Intelligence) ※音響関連のみ
  - カスタムキーワード検索（"audio", "speech", "sound", "music"等）
- **翻訳**: Google Translate API（英語→日本語）
- **更新頻度**: 1日1回（手動実行またはGitHub Actions）

### 2.2 検索・フィルタリング機能（優先度：高）

#### 2.2.1 日付フィルタ
- 期間指定（開始日〜終了日）
- プリセット（今日、過去7日間、過去30日間、過去1年間）

#### 2.2.2 キーワード検索
- 対象：タイトル（英語・日本語）、著者名
- リアルタイム検索（入力しながら絞り込み）

#### 2.2.3 カテゴリフィルタ
- arXivカテゴリ（cs.SD, eess.AS, cs.AI）での絞り込み
- 複数カテゴリの同時選択可能

#### 2.2.4 ソート機能
- 投稿日（新しい順・古い順）
- タイトル（50音順・アルファベット順）
- デフォルト：投稿日（新しい順）

### 2.3 ブックマーク機能（優先度：高）

#### 2.3.1 ブックマーク登録
- 論文一覧から任意の論文をブックマークに追加
- ブックマーク済みの論文には視覚的にマーク表示

#### 2.3.2 ブックマーク解除
- ブックマークから削除

#### 2.3.3 ブックマーク一覧表示
- ブックマークした論文のみを一覧表示
- 論文一覧機能の検索・フィルタリング機能も適用可能

### 2.4 タグ管理機能（優先度：高）

#### 2.4.1 タグの作成
- ユーザーが任意のタグ名を作成可能
- タグ名の例：「音声認識」「音楽情報処理」「信号処理」「深層学習」等
- タグの色をカスタマイズ可能（オプション）

#### 2.4.2 タグの付与
- ブックマークした論文に対して複数のタグを付与可能
- タグはブックマーク時または後から追加可能

#### 2.4.3 タグの削除
- 論文から特定のタグを削除
- タグ自体を削除（全論文から削除）

#### 2.4.4 タグによるフィルタリング
- 特定のタグが付いた論文のみを表示
- 複数タグによるAND/OR検索
  - AND: すべてのタグを持つ論文のみ
  - OR: いずれかのタグを持つ論文

#### 2.4.5 タグ一覧表示
- 作成済みタグの一覧を表示
- 各タグに紐づく論文数を表示

### 2.5 ユーザー認証機能（優先度：高）

#### 2.5.1 ログイン
- Googleアカウント認証（Firebase Authentication）
- メールアドレス/パスワード認証（オプション）

#### 2.5.2 ログアウト
- セッション終了

#### 2.5.3 ユーザーデータ管理
- ブックマークとタグはユーザーごとに独立して管理
- デバイス間でのデータ同期（Firebaseにより自動）

---

## 3. 非機能要件

### 3.1 パフォーマンス
- ページ読み込み時間：3秒以内
- 検索・フィルタリングのレスポンス：1秒以内

### 3.2 セキュリティ
- ユーザーデータはFirebaseのセキュリティルールで保護
- 他のユーザーのブックマーク・タグにアクセス不可

### 3.3 可用性
- GitHub Pagesによる24時間稼働
- データベース：Firebase（99.95%のSLA）

### 3.4 互換性
- 対応ブラウザ：Chrome、Firefox、Safari、Edge（最新版）
- レスポンシブデザイン（PC、タブレット、スマートフォン対応）

### 3.5 保守性
- コードの可読性を重視
- コンポーネント単位での開発（React）
- 環境変数によるAPI鍵管理

---

## 4. システム構成

### 4.1 フロントエンド
- **フレームワーク**: React
- **ホスティング**: GitHub Pages
- **UIライブラリ**: Material-UI または Tailwind CSS（検討中）

### 4.2 バックエンド
- **認証**: Firebase Authentication
- **データベース**: Firebase Firestore
- **ストレージ**: GitHub Pagesの静的ファイル（論文データJSON）

### 4.3 データ収集
- **言語**: Python
- **API**:
  - arXiv API（論文データ取得）
  - Google Translate API（翻訳）
- **実行環境**: ローカルまたはGitHub Actions

---

## 5. データ更新フロー

1. PythonスクリプトでarXiv APIから最新論文を取得
2. Google Translate APIでタイトルを日本語に翻訳
3. JSON形式でデータを保存（`data/papers.json`）
4. GitHub Pagesにデプロイ（自動または手動）
5. フロントエンドが最新のJSONを読み込み表示

---

## 6. 制約事項

- Twitter APIは使用しない（有料のため）
- arXiv APIのレート制限を遵守（1秒あたり1リクエスト推奨）
- Google Translate APIの無料枠を考慮（月50万文字まで）
- GitHub Pagesの容量制限（1GB推奨）
